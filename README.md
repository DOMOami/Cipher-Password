Encrypted password generation & management tool. Generate strong passwords → PBKDF2 + AES-256-GCM encrypt → local storage + Gmail backup → decrypt on mobile browser.

## Workflow

```
┌─ Sender (PC) ────────────────────────────────────┐
│                                                  │
│  password_generator.py                           │
│  Generate password → map to charset indices       │
│  → AES-256-GCM encrypt → save to password.txt    │
│  → Send via Gmail SMTP                           │
│  → Attachment: decrypt.html (obfuscated decryptor)│
│                                                  │
└───────────────┬──────────────────────────────────┘
                │  Email (Gmail)
                ▼
┌─ Recipient (Mobile) ─────────────────────────────┐
│                                                  │
│  Open attachment decrypt.html                     │
│  Enter unlock phrase → access decrypt page        │
│  Enter shared passphrase + paste ciphertext       │
│  → display plaintext password                     │
│                                                  │
│  All operations run locally in the browser         │
│                                                  │
└──────────────────────────────────────────────────┘
```

## Quick Start

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Generate config template
python password_generator.py
# First run creates config.json template, edit it as prompted

# 3. Edit config.json
# See configuration section below

# 4. Build the decryptor (set unlock passphrase, required before sending email)
python build_decrypt.py

# 5. Run again to generate a password and send email
python password_generator.py
```

## Configuration (config.json)

```json
{
  "sender_email": "your-account@gmail.com",
  "app_password": "Gmail app-specific password",
  "recipient_email": "recipient@gmail.com",
  "passphrase": "shared passphrase agreed with recipient"
}
```

**How to get a Gmail app-specific password:**
1. Enable [2-Step Verification](https://myaccount.google.com/security)
2. Go to [App Passwords](https://myaccount.google.com/apppasswords)
3. Select "Mail" → "Other" → generate a 16-character password (remove spaces when pasting)

## Security Design

| Aspect | Approach |
|--------|----------|
| Encryption | PBKDF2 (SHA-256, 600,000 iterations) → AES-256-GCM (authenticated encryption) |
| Password in transit | Only ciphertext is sent; plaintext never appears in email |
| decrypt.html | Core JS encrypted with AES-256-GCM; unrecoverable without unlock passphrase |
| Local storage | password.txt stores ciphertext only |
| Browser security | Web Crypto API; keys never leave the browser |

## File Structure

```
├── password_generator.py   # Main program
├── build_decrypt.py        # Obfuscated decrypt.html builder
├── decrypt.html            # Mobile decrypt page (generated by build_decrypt.py)
├── config.json.example     # Configuration template
├── requirements.txt        # Python dependencies
└── .gitignore              # Protects sensitive files
```

Generated at runtime (not tracked by Git):
- `config.json` — email and passphrase config
- `password.txt` — encrypted password records
- `decrypt.html` — mobile decrypt page (generated by build_decrypt.py)

## Dependencies

- Python 3.8+
- [cryptography](https://cryptography.io/) — PBKDF2, AES-256-GCM

└───────────────┬──────────────────────────────────┘
                │  邮件 (Gmail)
                ▼
┌─ 接收方 (手机) ──────────────────────────────────┐

# 1. 安装依赖
pip install -r requirements.txt

python password_generator.py

## 配置 config.json

```json
{
  "sender_email": "你的邮箱@gmail.com",
  "app_password": "Gmail应用专用密码",
  "recipient_email": "接收密码的邮箱@gmail.com",
  "passphrase": "你和接收者约定的共享口令"
}
```

**Gmail 应用专用密码获取：**
1. 开启 [两步验证](https://myaccount.google.com/security)
2. 前往 [应用专用密码](https://myaccount.google.com/apppasswords)
3. 选择"邮件" → "其他" → 生成 16 位密码（填入时去掉空格）

## 安全设计

| 环节 | 方案 |
|------|------|
| 加密算法 | PBKDF2 (SHA-256, 60万次迭代) → AES-256-GCM（认证加密） |
| 密码发送 | 仅发送密文，明文永不出现在邮件中 |
| decrypt.html | 核心JS经 AES-256-GCM 加密，解锁口令错误无法解密出算法 |
| 本地存储 | password.txt 仅存密文 |
| 浏览器安全 | Web Crypto API，密钥不离开浏览器 |
